# 🔰 Guide Débutant : Les Super-Pouvoirs de l'IA

Bienvenue ! Ce guide vous explique comment l'IA analyse vos documents comme un véritable ingénieur système.

---

## 1. Les 3 "Micro-Agents" (Votre équipe d'experts)

Imaginez que vous avez trois experts qui travaillent à la chaîne sur chaque phrase de votre document :

### 📖 A. Le Traducteur (Agent BABOK)
*   **Son rôle** : Rendre les phrases claires.
*   **Action** : Si le client écrit "L'accès doit être sécurisé", le traducteur réécrit : "Le Système [Qui ?] doit Authentifier [Quoi ?] via un Mot de passe [Comment ?]".
*   **Utilité** : On sait exactement qui doit faire quoi.

### 🐺 B. Le Radar à Loups (Agent de Désambiguïsation)
*   **Son rôle** : Traquer les pièges.
*   **Action** : Il cherche les mots "flous" comme *rapide*, *moderne*, *simple*, *efficace*.
*   **Utilité** : Il vous alerte : "Attention, ce mot est dangereux car il n'est pas chiffré. Le client pourra dire que ce n'est jamais assez rapide !".

### 🛡️ C. Le Gardien (Agent de Complétude ISO 25010)
*   **Son rôle** : Deviner ce qui manque.
*   **Action** : Si vous parlez de "Stocker des fichiers", il demande immédiatement : "Et comment on les supprime ? Comment on les sauvegarde ?".
*   **Utilité** : Il évite d'oublier des fonctionnalités obligatoires mais non écrites.

---

## 2. Comment lire les résultats ?

Après avoir lancé l'**Audit Granulaire**, ouvrez le fichier `data/granular_audit_report.md`.
- **Statut VALIDATED** : La phrase est claire et précise.
- **Statut PENDING_CLARIFICATION** : L'IA a détecté un "loup". Vous devez poser une question au client.

---

## 3. Pourquoi est-ce important ?
En phase de réponse à un appel d'offres, l'ambiguïté est votre ennemie. Ce système vous permet de transformer un document flou en une liste de tâches techniques précises et chiffrables.
